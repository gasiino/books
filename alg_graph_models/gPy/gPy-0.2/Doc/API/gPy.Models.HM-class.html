Hierarchical models

Probability distributions which are the product of a number of factors. If each factor is a Parameters.Factor object then we have a fitted model, i.e. a particular probability distribution. If one or more factor is not a Parameters.Factor object but only a Parameters.SubDomain object (and thus lacks data), then a (non-singleton) set of distributions is defined.

add(self, factor)

source code 

Include a factor in a hierarchical model

If a factor with exactly the same variables already exists in the model, then this factor is multiplied by factor. This is to ensure that there is a 1-1 mapping between factors and hyperedges (recall that the associated HyperGraph is a set of hyperedges). Otherwise factor is just added to the existing factors.
Parameters:
  • factor (Factor object) - The factor to be added

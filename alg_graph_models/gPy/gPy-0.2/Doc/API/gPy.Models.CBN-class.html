__init__(self, factors=(), domain=None, adg=None)
(Constructor)

source code 

BN constructor. Each factor has its domain expanded to be the domain of the model, if necessary.

If factors is a sequence of factors then each is merely assumed to be a CPT object, but there is no check done.

Parameters:
  • factors - The factors in the hierarchical model. (Alternatively an existing object of class FR (or its subclasses), in which case self has identical attributes, except possibly the attribute specifying its adg.)
  • domain - A domain for the model. If None the internal default domain is used.
  • new_domain_variables - A dictionary containing a mapping from any new variables to their values. domain is updated with these values
  • must_be_new - Whether domain variables in new_domain_variables have to be new
  • check - Whether to check that all variables exist in domain
  • adg - The acyclic digraph for the BN. This is simply assumed to be the correct ADG, no check is made. If adg is not supplied the correct ADG is created if possible, otherwise a DirectedCycleError is raised.
Raises:
  • VariableError - If a variable in new_domain_variables already exists with values different from its values in new_domain_variables; Or if must_be_new is set and the variable already exists. Or if check is set and a variable in variables is not in the domain
  • AttributeError - If adg was not supplied and any of the supplied factors are not Parameters.CPT objects.
  • DirectedCycleError - If adg was not supplied and there is no ADG consistent with the CPTs provided.

from_adg_data(adg, data, prior=1)
Static Method

source code 

Construct a CBN from an ADG and estimates of the parameters from some observations.

Parameters:
  • prior - the Dirichlet prior parameter (the same parameter value is used for all instances!) Note there may be some problems with this method: a different prior is used by the BDeu score. However, in practice, for parameter estimation, this prior method seems to be ok. I was lazy and it was simple to implement (cb). If prior is zero, then the parameters are the maximum likelihood estimation solutions.

extend_to(self, other)

source code 

Extend the domains of each of the variables in other to the corresponding values. The parameters corresponding to values introduced to the domains are set to zero. When conditioning, BN removes elements from the domains of variables. Sometimes it is useful to add the zeroes e.g. when using an additive binary operator.

Parameters:
  • other - a dictionary mapping variables to their domains. Note the current domain must be a subset of the new domain.

good_sets_forced_variables(self)

source code 

Return the good independent sets of variables to force. This is an approximation since ideally you want the edge which implies the most propagation when resolved. The approximation will resolve all edges. However some forcing may be unnecessary.

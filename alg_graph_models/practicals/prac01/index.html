Finding the information

Good news! You do not need to memorise how Python works. Lots of documentation is available, reachable via a variety of routes. Firstly there is the documentation that comes with the Python distribution. This is locally available in HTML format at /usr/doc/python-2.5.2/html/Python-Docs-2.5.2/index.html Referring to locally installed documentation is better since it cuts down on unnecessary internet traffic and ensures that you are reading the documentation for the version of Python that you are using. The documentation for the most recent version of Python is always available from the official Python site. Just look for the documentation links on the left hand side (in particular the Beginner's Guide link).

You can also get documentation directly from the Python interpreter. Just start up the Python interpreter by typing python at the command line. Then type help() and follow the instructions. Typing Control-d stops gets you out of the help system and another Control-d will stop the interpreter.

Getting started with the Python interpreter

To start the Python interpreter, just type python at the Unix command line. To finish with it just enter control-d.

Hello world!

Let's get this out of the way. The Python 'Hello world' program is a single line: print "Hello world!". Save this in a file called, say, hello.py Since the print statement is at the top-level, there should be no identation. Run the program with: python hello.py

You can compare your Python "Hello world!" program to the same program written in a host of other languages at the Wikipedia.

Checking that gPy works

By following the instructions, ensure your PYTHONPATH is set so that your programs can find the gPy package. Now grab this program and check it works. You might also want to fetch the demo programs for the lectures so far, and check they work. (Some of these programs require input files, make sure you get these also.)

The Python tutorial

Well, the claim is "learn Python in an afternoon". Let's put this to the test. Just work your way through the Python tutorial at /usr/doc/python-2.5.2/html/Python-Docs-2.5.2/tut/tut.html, using the interpreter to try out the examples. On a first pass, you can skip Sections 1, 2, 3.1.3, 4.7, 6, 7, 8.4-8.7, 9.7-9.11. You will need to check out at least Sections 6 and 7 on a second pass.
